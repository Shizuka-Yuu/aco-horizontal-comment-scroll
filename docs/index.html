ACO Horizontal Comment Scroll (v1.0.0)

某動画サイト風にコメントを流すAbemaTV用ブラウザ拡張機能

※コメントが流れる様子のデモ動画

概要・機能

AbemaTVの視聴ページにて、画面を広く使いながらコメントを楽しめるようにする非公式拡張機能です。

  • 某動画サイト風コメント流し: 動画プレイヤー上にコメントを右から左へ流します。
  • DOM監視モード: 公式コメント欄のDOMを監視してコメントを抽出します。（※推奨モード）
  • API監視モード: コメント欄を閉じてプレイヤーを最大化している場合、APIへ直接リクエストを送ってコメントを取得します。
  • 描画制御: 数式ベースの衝突回避アルゴリズムにより、コメントが重ならないよう動的にレーンを決定します。
  • 厳密なキューイング仕様:
    ・鮮度チェック: 各コメントに投入時のタイムスタンプを付与し、30秒以上経過した古いコメントを自動破棄することで、ページリロード時、一時停止後などのキュー溜まりによる大量放出（コメントの洪水）を防止します。
    ・無制限の流速保持: 通常視聴時はコメントを一切破棄せず、理論上無限のキュー待機を許容します。これは、コメントを一つも漏らさず表示することこそが、ライブ視聴体験として最も誠実であるという設計思想によるものです。
    ・優先度制御: システムメッセージはキューの先頭へ即座に割り込みを行い、視聴者に重要な通知を逃さず届けます。
  • 公式ブロックとの連携: 公式機能でブロックしたユーザーは、本拡張機能のオーバーレイからも即座に非表示となります。
  • オプション設定: 拡張機能の詳細ページから必要最小限の設定変更が可能です。
  • NGフィルター: NGワード・文字数・正規表現によるフィルター設定が可能です。
  • ニュースチャンネル対応: 通常チャンネルとはバックエンドの仕様が異なる為、DOM監視モードを前提としています。コメント欄が閉じられている場合は一時的に適切なシステムメッセージが流れます。

※オーバーレイ自体の表示ON/OFFは、プレイヤー右下に追加された専用トグルボタンでいつでも切り替え可能です。

マニュアル導入手順

Google Chrome / Edge

  1. ZIPファイルを解凍します。
  2. ブラウザの拡張機能管理ページ（chrome://extensions）を開きます。
  3. 右上の「デベロッパーモード」をONにします。
  4. 「パッケージ化されていない拡張機能を読み込む」をクリックし、解凍したフォルダを選択します。
注意: 開発者モードの警告が表示されることがありますが、「無効にする」ボタンは押さないでください。押すと機能が停止します。

Firefox

  1. ZIPファイルを解凍します。
  2. ブラウザのアドレスバーに about:debugging と入力し移動します。
  3. 左側のメニューから「このFirefox」をクリックします。
  4. 「一時的な拡張機能を読み込む...」をクリックし、フォルダ内の manifest.json を選択します。
注意: Firefoxの「一時的な拡張機能」は、ブラウザを閉じると設定がクリアされます。再起動のたびに上記の手順で再読み込みしてください。

プライバシー・セキュリティ・権限について

本拡張機能は、ユーザーのプライバシーと安全を最優先に設計されており、プログラムのソースコードはすべてGitHubにてオープンソース（MIT License）で開示されています。

  • 必要な権限と使用目的:
    ・manifest_version: 3 (最新のセキュアな拡張機能仕様) を採用しています。
    ・*://abema.tv/* (content_scripts): 配信ページ上にコメントを重ねて描画するために使用します。
    ・storage (permissions): ユーザーの設定値（フォントサイズやNGワードなど）をブラウザ内部に保存・同期するために使用します。
    ・webRequest & https://*.akamaized.net/* (host_permissions): プレイヤーが読み込むメディアセグメントのURL変化をローカルで監視し、番組スロットIDの切り替え（番組改編）を自動検知するために使用します。
    ・https://api.p-c3-e.abema-tv.com/* (host_permissions): コメント欄を閉じた状態（API監視モード）の時に、公式コメントAPIからデータを直接フェッチするために使用します。
  • データ通信（ローカル完結設計）:
    API監視モード時のみ、AbemaTVの公式コメントAPIに対して直接リクエストを行います。また、番組切り替え検知用の動画セグメントURL監視もブラウザ内部（ローカル）でのマッチング処理のみで完結しています。
    これらで取得・検知した情報をアドオン作者や第三者の外部サーバーに送信（情報収集）することは一切ありません。
  • データ収集: 閲覧履歴、Cookie、パスワードなどの個人情報を取得・送信・保存することは一切ありません。すべてブラウザ内部のメモリおよびローカルストレージでの処理で完結します。

アンインストール手順

本アドオンを不要になった場合は、以下の手順で完全に削除してください。

  • ブラウザからの削除:
    • Chrome/Edge: 拡張機能管理ページ（chrome://extensions）を開き、本アドオンの「削除」ボタンをクリックしてください。
    • Firefox: 拡張機能管理ページ（about:addons）を開き、本アドオンの横の「…」から「削除」を選択してください。
  • ディレクトリの削除: 本アドオン導入時に作成した解凍後のフォルダ（アドオン本体）を、PC上の保存先から手動で削除してください。

ブラウザの機能削除に加え、ディレクトリを手動削除することでPC内から完全に削除されます。
これ以外にシステム設定ファイル等が残ることはありません。

💡 動作がおかしいときは（トラブルシューティング）

アドオンの挙動が不安定になった場合は、以下の手順を上から順にお試しください。

  1. アドオンの再読み込み: 拡張機能管理画面（Chrome系は更新アイコン、Firefoxは「再読み込み」）をクリック.
  2. ページのリロード: 現在視聴中のタブを再読み込み（F5キー）。
  3. ブラウザの再起動: ブラウザを一度完全に終了し、再起動してください。

    ※Firefoxの場合は、再起動後に再度「一時的な拡張機能を読み込む」手順を行ってください。

既知のバグ

コメントブロックを開くと右端から左側にブロックがせり出す、この時とコメントの排出が重なることで
スクリーン領域が一瞬で変化することによるコメントの衝突（重なり）が発生する事がある。
仕様上モードの切り替えを告知するシステムメッセージが衝突しやすい。

コメントオーバーレイの表示・非表示トグル用、拡張機能専用ボタンが押せなくなることがある。（特にChrome環境下）
公式のSPAによるレンダリングの上書きや既存要素の破壊によってボタンがゴースト化することに起因する。
ウインドウを一度最大化して戻す等の操作（レンダリングの強制）を行えば多分押せるようになったりする。

運用に関する考え方（DOM監視とAPI監視について）

本アドオンにはDOM監視とAPI監視の2つのモードがあります。公式側の描画負荷（DOM生成プロセス）と、拡張機能によるAPIポーリング負荷（ネットワークリクエスト）のどちらが軽量かは状況に依存します。

そのため、可能な限り公式コメント欄を展開した状態での「DOM監視モード」での視聴を推奨いたします。こちらは配信サービス本来の仕組みを尊重し、不要な通信を避ける最も安定した視聴スタイルであると考えています。API監視は、あくまでプレイヤーの没入感を優先し、コメント欄を閉じたい場合の代替手段としてご利用ください。
